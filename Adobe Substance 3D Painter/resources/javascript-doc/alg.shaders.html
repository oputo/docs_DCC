 Configuration for setting data

If no project opened

Examples

     alg.shaders.setParameters(0, {
       colors: [0, 0, 0],
       itensity: 0.5
     });

     
// Compute a variable which goes from 0 to 1 in 1 minute
     var date = new Date();
     var animationValue = (date.getSeconds() + date.getMilliseconds() / 1000.0) / 60.0;

     // Set this variable as an animation parameter to each shader instances
     var shaders = alg.shaders.instances();
     for (var i in shaders) {
       alg.shaders.setParameters(shaders[i].id, {
         animation: animationValue
       });
     }

TransactionOptions

Configuration for setting data
Type:
  • Object
Properties:
Name Type Argument Default Description
undoable bool <optional>
 true Mark changes as undoable by the user. It is relevant for a change to be undoable as example if we reimplement a new GUI to set values. On the contrary, if we manipulate parameters at a given frequency to create an animation, we should pay attention to the undoable behavior.
